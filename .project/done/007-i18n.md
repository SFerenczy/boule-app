# Ticket: Internationalization (i18n)

## Priority

High

## Goal

Support German, French, and English in the app using Paraglide.js, so players in Germany and France can use it in their language.

## Context

The app is used in Germany and France. The UI is currently English-only with some German domain terms (Legen, Schießen). Boule/petanque is big in both countries — the app needs to speak the user's language.

Paraglide.js (by Inlang) is the best fit for this stack: fully typesafe, tree-shakeable, Svelte 5 first-class support, works with static adapter. Messages compile to typed functions — missing keys break the build, not the UI.

## Acceptance

- [x] App renders in German when browser language is `de`, French for `fr`, English for everything else
- [x] All user-facing strings come from Paraglide message functions — no hardcoded display text in components
- [x] Language choice persists across sessions (localStorage)
- [x] Missing translation key = build error (Paraglide default)
- [x] Static adapter still works
- [x] `just check` passes
- [x] No visible language switcher during active gameplay — settings only

## Deviations

- **No URL-based language routing.** The ticket originally specified `/de/`, `/fr/`, `/en/` URL prefixes. This was dropped because the app is a single-route SPA — URL routing would require enabling SSR, adding SSR guards for Dexie, and prerendering locale variants, all for zero user benefit in a PWA. See DEC-007.
- **Package:** Used `@inlang/paraglide-js` (v2) instead of the deprecated `@inlang/paraglide-sveltekit`.

## Dependencies

- None (can be done on current codebase)

## Notes

- Default locale: `en`. German and French are additional locales.
- Keep domain terms consistent: "Legen" and "Schießen" stay as-is in German, translate to descriptive equivalents in English/French (e.g. "Pointing"/"Shooting", "Pointage"/"Tir").
- Paraglide supports ICU MessageFormat for plurals — use it where needed (e.g. "1 game" vs "3 games").
